## 8. Mermaid Diagrams (for GitHub rendering)

### Research Flow

```mermaid
flowchart TD
    A[Admin Creates Research Task] --> B{Select Data Sources}
    B --> C[Google Places API]
    B --> D[Web Scraping]
    B --> E[Manual Seeds]

    C --> F[Raw Results]
    D --> F
    E --> F

    F --> G[Claude Analysis]
    G --> H[Extract Business Info]
    G --> I[Calculate Confidence Score]
    G --> J[Detect Duplicates]

    H --> K[Create Provider Lead]
    I --> K
    J --> K

    K --> L{ABN Lookup}
    L -->|Found| M[Mark ABN Verified]
    L -->|Not Found| N[Flag for Manual Check]

    M --> O[Lead Ready for Review]
    N --> O

    O --> P{Admin Review}
    P -->|Approve| Q[Queue Outreach]
    P -->|Reject| R[Archive Lead]
    P -->|Duplicate| S[Merge with Existing]
```

### Outreach Flow

```mermaid
sequenceDiagram
    participant A as Admin
    participant S as System
    participant C as Claude
    participant E as Email (Resend)
    participant P as Provider

    A->>S: Approve Lead
    S->>C: Generate Parent Persona Email
    C->>S: Return Email Draft
    S->>A: Show Email for Approval
    A->>S: Approve (with optional edits)
    S->>E: Send Email
    E->>P: Deliver Email

    alt Provider Responds
        P->>E: Reply Email
        E->>S: Webhook: Reply Received
        S->>C: Analyze Response
        C->>S: Positive/Negative/Question
        S->>A: Notification
    else No Response (5 days)
        S->>C: Generate Follow-up
        C->>S: Follow-up Draft
        S->>A: Show for Approval
    end
```

### Term Confirmation Flow

```mermaid
stateDiagram-v2
    [*] --> Pending: Term starts
    Pending --> ReminderSent: 4 weeks before end

    ReminderSent --> Confirmed: Provider confirms
    ReminderSent --> ReminderSent: Follow-up (2 weeks)
    ReminderSent --> NotResponding: No response after 3 attempts

    Confirmed --> [*]: Term ends
    NotResponding --> ManualReview: Admin escalation
    ManualReview --> Inactive: Mark provider inactive
    ManualReview --> Confirmed: Manual confirmation
```
